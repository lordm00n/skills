# Wikis Create - 新增文档规范

本 subskill 规范如何在 wikis 目录中新增文档。

## 执行前准备

1. **确认 wikis 路径** - 用户指定或默认 `~/wikis`
2. **确认文档存放位置** - 根据内容类型判断放入哪个子目录
3. **检查目录结构** - 确认目标子目录是否存在

## 文件命名规范

### 核心原则

文件名应该**语义化**，能够清晰表达文档内容：

| 好的命名 | 差的命名 | 原因 |
|----------|----------|------|
| `golang.md` | `g.md` | 语义不明确 |
| `python-virtualenv.md` | `py-venv.md` | 英文完整单词更清晰 |
| `git-worktree-guide.md` | `git-wt.md` | 缩写不易理解 |

### 命名模式

```
<主题关键词>.md
```

- 使用英文或拼音（保持一致）
- 使用连字符 `-` 分隔多个词
- 避免无意义的编号如 `doc1.md`、`note2.md`

## 目录组织规范

### 子目录创建规则

- **最多一层子目录** - wikis 下可直接创建子目录，但子目录下不能再嵌套
- **必须有 index.md** - 每个子目录必须包含 `index.md` 作为索引

### 目录创建流程

1. 检查目标目录是否存在
2. 如果不存在，创建目录
3. 检查目录中是否有 `index.md`
4. 如果没有，创建 `index.md` 并说明如何查找该目录下的文档
5. 更新父目录的 `index.md`（如果需要）

## 新增文档流程

### Step 1: 确定存放位置

根据内容类型选择目录：

| 内容类型 | 存放目录 | 示例 |
|----------|----------|------|
| 环境配置 | `env/` | `env/golang.md` |
| 学习笔记 | `notes/` | `notes/design-patterns.md` |
| 工具手册 | `tools/` | `tools/vim.md` |
| 项目文档 | `projects/` | `projects/myapp.md` |

如果不确定，询问用户。

### Step 2: 创建文档

1. 创建文件，写入内容
2. 内容使用 Markdown 格式
3. 添加文件创建说明

### Step 3: 更新索引

创建或更新相关 `index.md`：

1. 更新子目录的 `index.md`，添加新文档链接
2. 如果是新建的子目录，同时更新 `wikis/index.md`

## 示例

### 示例 1: 创建新分类目录

**场景**: 用户要创建一个 `tools` 目录存放工具手册

**操作**:
```bash
mkdir -p ~/wikis/tools
```

**创建 tools/index.md**:
```markdown
# 工具手册

> 本目录存放常用工具的使用手册。

## 文档列表

| 文档 | 说明 |
|------|------|
```

**更新 wikis/index.md** - 添加 `tools/` 条目

### 示例 2: 在现有目录新增文档

**场景**: 用户要创建 `env/rust.md` 记录 Rust 环境配置

**操作**:
1. 确认 `env/` 目录存在
2. 创建 `env/rust.md`
3. 更新 `env/index.md` 添加 rust.md 链接

## 检查清单

创建文档后，确认：

- [ ] 文件名语义化
- [ ] 内容格式正确 (Markdown)
- [ ] 相关 index.md 已更新
- [ ] 目录结构符合规范（最多一层子目录）
