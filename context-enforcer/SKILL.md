## 核心原则

**每次用户消息都会触发这个 skill。** 不要跳过，不要判断"这个任务太简单不需要索引"。上下文加载是廉价的，遗漏上下文是昂贵的。

## 工作流程

### 1. 立即读取 AGENTS.md

接收到用户消息后，第一时间读取项目根目录的 `AGENTS.md`（或 `CLAUDE.md`、`.claude.md`）。

不要等到需要上下文时才读——在分析任务之前就读。

### 2. 解析声明的上下文路径

从 AGENTS.md 中提取所有上下文路径：

#### 2.1 本地仓库外上下文

在 `## 本地仓库外上下文` 章节找到所有路径列表，逐个读取对应的 `index.md` 文件。

```
示例格式：
- ~/wikis/index.md
- ~/docs/project-notes/index.md
```

#### 2.2 远端文档索引

1. 找到 `## 远端文档` 章节中的索引文件路径（通常是 `.agents/remote-docs-index.md`）
2. 读取该索引文件
3. 对于每个远端文档条目，读取其内容或获取方式

#### 2.3 代码约束

1. 在 `## 代码约束` 章节找到约束目录路径（通常是 `.agents/constrains/`）
2. 列出该目录下的所有 `.md` 文件
3. 读取所有约束文件

#### 2.4 仓库内上下文

在 `## 仓库内特定的上下文` 章节找到所有路径，读取对应的 `index.md` 或 `index.md` 指向的文档。

### 3. 构建上下文摘要

在开始分析用户任务之前，先在内心（或输出中）构建一个简短的上下文摘要：

```
已加载的上下文：
- 本地文档: [路径列表]
- 远端文档: [数量] 个文档
- 代码约束: [约束文件列表]

现在开始分析用户任务...
```

### 4. 决定是否需要进一步索引

根据用户任务的具体内容，决定是否需要额外索引：

| 任务类型 | 是否需要进一步索引 |
|---------|-------------------|
| 修改/查看某文件 | 读取该文件 |
| 涉及某个子目录 | 读取该目录的 index.md |
| 提到特定约束 | 读取对应约束文件 |
| 使用某技术/框架 | 读取相关远端文档 |

如果需要额外索引，立即执行，不要等到后续步骤。

### 5. 执行用户任务

在完整上下文的支撑下，执行用户的任务。

## 常见问题

**Q: 每次都读这些文件会不会太慢？**
A: 这些通常是小的索引文件（几百字节到几 KB），读取速度很快。真正耗时的远端文档只在需要时才获取。

**Q: 任务太简单，还需要读 AGENTS.md 吗？**
A: 是的。"2+2 等于几"可能不需要，但用户的消息通常涉及项目上下文。养成习惯比偶尔遗漏好。

**Q: AGENTS.md 不存在怎么办？**
A: 建议用户创建，或询问他们项目的上下文应该在哪里。

## 与 context-updater 的配合

这个 skill 负责"使用"现有上下文索引。`context-updater` 负责"维护"索引。当发现上下文缺失或过时时，触发 context-updater 来更新索引。
