## 工作流程

### 1. 解析用户提示词

从用户的提示词中提取可能的上下文引用：

- **远程文档链接**：URL、链接形式的文本（如 `https://...`、`docs/...`、`[标题](链接)`）
- **本地文档路径**：文件系统路径（如 `~/docs/...`、`/path/to/...`、相对路径）
- **编码约束说明**：包含约束性质的描述（如 "不要用 X"、"应该用 Y"、"禁止 Z"、"必须 Z"）

### 2. 检查上下文是否已索引

读取 AGENTS.md，找到各部分索引文件/目录的路径，然后逐项检查：

#### 2.1 检查本地文档路径

在 AGENTS.md 中找到"本地仓库外上下文"章节，提取已索引的路径列表。

#### 2.2 检查远程文档链接

1. 在 AGENTS.md 中找到远端文档索引文件路径（`.agents/remote-docs-index.md`）
2. 读取该索引文件
3. 检查用户提到的远程链接是否已被索引

#### 2.3 检查编码约束

1. 在 AGENTS.md 中找到编码约束目录路径（`.agents/constrains/`）
2. 列出该目录下的所有 `.md` 文件
3. 在这些文件中搜索用户提到的编码约束内容

### 3. 汇总检查结果

将检查结果整理为以下格式：

```
已索引的上下文：
- 本地路径: [列表]
- 远程链接: [列表]
- 编码约束: [列表]

缺失的上下文：
- 本地路径: [用户提到但未索引的]
- 远程链接: [用户提到但未索引的]
- 编码约束: [用户提到但未索引的]
```

### 4. 询问用户是否更新

如果存在缺失的上下文，向用户确认：

```
检测到以下上下文未被索引：
- [缺失项列表]

是否需要更新索引？
```

### 5. 执行更新（如果用户确认）

根据用户的选择，更新对应的索引：

#### 5.1 添加本地文档路径

- 在对应路径下创建 `index.md`（如果不存在）
- 在 AGENTS.md 的"本地仓库外上下文"章节中添加路径

#### 5.2 添加远程文档链接

- 读取 `.agents/remote-docs-index.md`
- 添加新的远程文档条目（格式由用户或 remote-docs-updater skill 定义）

#### 5.3 添加编码约束

- 在 `.agents/constrains/` 目录下创建或更新对应的 `.md` 文件
- 命名格式：`<主题>.md`
