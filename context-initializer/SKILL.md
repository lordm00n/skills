description: 用于初始化项目的上下文入口索引。当你需要为项目设置 Agent 可访问的上下文结构、创建 AGENTS.md 主索引文件、或向现有 AGENTS.md 添加新的上下文路径时使用。包括：创建项目级索引入口、指定代码约束目录、配置远端文档索引、以及添加仓库内外的上下文路径。注意：如果用户只是简单地说"初始化项目"，这很可能意味着他们需要这个 skill。
---

## 工作流程

### 1. 询问上下文路径

向用户询问以下信息：

1. **仓库内的上下文路径**：项目仓库内的目录列表（可以是多个），每个路径都需要有 `index.md` 文件。如果 `index.md` 不存在，创建一个空文件。

2. **本地仓库外的上下文路径**：项目仓库外的目录列表（可以是多个），每个路径都需要有 `index.md` 文件。如果 `index.md` 不存在，创建一个空文件。

### 2. 创建必需的目录和文件

按顺序执行以下操作：

1. **创建 `.agents/` 目录**（如果不存在）

2. **创建远端文档索引**：
   - 路径：`.agents/remote-docs-index.md`
   - 如果文件不存在，创建一个空文件
   - 在 AGENTS.md 中添加对该文件的引用

3. **创建约束目录**：
   - 路径：`.agents/constrains/`
   - 如果目录不存在，创建该目录

### 3. 创建或更新 AGENTS.md

**如果 AGENTS.md 不存在**，创建并写入以下基础结构：

```markdown
# Agent Context Index

## 项目信息
- 项目根目录: {当前工作目录}

## 仓库内特定的上下文
<!-- 用户提供的仓库内路径会添加到这里 -->
<!-- 项目开发过程中产生的 specs/文档/plan -->

## 本地仓库外上下文
<!-- 用户提供的仓库外路径会添加到这里 -->

## 远端文档
<!-- 可以通过读取索引文件来查找远端上下文的获取方式，如需要获取远端上下文，则读取该索引文件 -->
- 索引文件: [.agents/remote-docs-index.md](.agents/remote-docs-index.md)

## 代码约束
- 约束文档存放在 [.agents/constrains/](.agents/constrains/) 目录下
- 命名格式：`<主题>.md`
- 需要时使用 `ls .agents/constrains/` 或 `grep` 命令搜索相关约束文件
```

**如果 AGENTS.md 已存在**，将其更新为包含上述所有章节（保留现有内容，将新章节添加到合适位置）。

### 4. 添加上下文路径到 AGENTS.md

对于用户提供的每个路径（仓库内和仓库外）：

1. 检查 `{path}/index.md` 是否存在，如果不存在则创建空文件
2. 将该路径作为相对路径添加到 AGENTS.md 的对应章节中：
   - 仓库内路径 → "## 仓库内上下文" 章节
   - 仓库外路径 → "## 本地仓库外上下文" 章节

使用以下格式添加：

```markdown
- {相对路径}/index.md
```

注意：
- 仓库内路径使用相对于项目根目录的路径
- 仓库外路径使用绝对路径（保持原样）

### 5. 确认完成

告诉用户已完成的工作：
- 创建/更新的文件列表
- 每个上下文路径对应的 index.md 是否已创建
