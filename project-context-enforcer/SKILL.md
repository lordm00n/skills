# Project Context Enforcer

这个 skill 用于强制 Agent 在每次处理任务时都索引 AGENTS.md 中声明的上下文，确保 Agent 不会遗漏重要的项目信息。

## 工作流程

### 步骤 1：读取 AGENTS.md

当这个 skill 被调用时，首先读取项目根目录下的 `AGENTS.md` 文件。这个文件包含了项目中所有上下文索引的路径。

### 步骤 2：索引所有上下文路径

解析 AGENTS.md 文件，提取所有被索引的上下文文件/目录路径：

- **远程文档索引**：`.agents/remote-docs-index.md`
- **项目文档索引**：`.agents/project-docs-index.md`
- **项目外本地知识库**：AGENTS.md 中列出的路径
- **编码约束**：`.agents/constraints.md`

### 步骤 3：读取被索引的上下文文件

读取步骤 2 中列出的所有索引文件，获取其中包含的具体文档/目录路径。
- **远程文档索引**：`.agents/remote-docs-index.md`
  - remote-docs-index.md 会包含文档的 url 以及描述等信息，你可以直接通过这些信息来判断后续是否需要进一步读取
- **项目文档索引**：`.agents/project-docs-index.md`
  - project-docs-index.md 会包含项目内文档的**目录路径**，而不是具体的文档以及描述，你需要用 `ls` 命令来列出这些目录下的文档，然后通过文档的名称/主题，来判断后续是否需要进一步读取
- **项目外本地知识库**：AGENTS.md 中列出的外部文档路径
  - 项目外本地知识库的索引文档路径，包含了对知识库的结构的说明，你需要读取该索引文档，根据知识库的结构，判断后续是否需要进一步读取
- **编码约束**：`.agents/constraints.md`
  - constraints.md 会包含项目的编码规范，你需要读取该文件，根据其中的规范，判断后续是否需要使用

### 步骤 4：根据用户提示选择相关上下文

检查用户的提示词/需求，判断需要哪些上下文：

- **如果提示涉及远程文档**：读取 `.agents/remote-docs-index.md` 中相关的远程文档
- **如果提示涉及项目内文档**：根据 `.agents/project-docs-index.md` 中索引的项目文档目录路径，读取具体的相关文档
- **如果提示涉及外部知识库**：根据知识库的索引文档路径，读取知识库的相关文档
- **如果提示涉及编码规范**：读取 `.agents/constraints.md` 中的相关约束

### 步骤 5：加载上下文内容

对于判断为相关的上下文，读取其实际内容，确保在处理用户任务前已加载完整的相关上下文。

## 重要提示

- **始终激活**：每次用户提出需求时都要执行这个 skill
- **先索引后行动**：确保在开始处理任务前完成所有上下文的索引
- **按需读取**：不需要读取所有索引文档，只读取与当前任务相关的
- **保持上下文**：将读取的上下文信息保留在当前会话中供后续使用
