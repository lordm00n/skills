# Project Context Initializer

这个 skill 用于初始化项目中的上下文索引结构，帮助建立项目的上下文管理基础设施。

## 工作流程

### 步骤 1：创建 AGENTS.md 主索引文件

首先在项目根目录下创建 `AGENTS.md` 文件，作为其他上下文索引文件的引用点。使用以下模板：

```markdown
# 上下文索引文件

本文件是项目中的上下文索引文件，包含了有哪些上下文，以及其对应的索引文件路径。

## 远程文档索引
// 指向远程文档的索引文件，包含了有哪些远程上下文，以及其获取方式。

## 项目文档索引
// 指向项目内的文档的索引文件，包含了有哪些项目内的上下文。

## 项目外，但在本地的知识库文档索引
// 指向项目外，但在本地的知识库文档的索引文件，包含了有哪些不在项目内的知识库文档。

## 项目内的编码约束
// 指向项目内的编码约束文件，包含了有哪些项目内的编码约束。
```

### 步骤 2：创建远程文档索引文件

创建 `.agents/remote-docs-index.md` 文件：

1. 直接创建这个文件
2. 在文件中添加描述："这是远程文档的索引文件，包含了有哪些远程上下文，以及其获取方式"
3. 在 `AGENTS.md` 中的"远程文档索引"部分添加引用：
   ```markdown
   ## 远程文档索引
   - 远程文档索引: .agents/remote-docs-index.md
   ```

### 步骤 3：创建项目文档索引文件

创建 `.agents/project-docs-index.md` 文件：

1. 直接创建这个文件
2. 在文件中添加描述："这是项目内的文档的索引文件，包含了有哪些项目内的上下文"
3. 询问用户项目内需要索引的目录有哪些，例如："当前项目中需要索引的上下文目录有哪些？"
4. 根据用户提供的目录，扫描并总结对应的目录结构，补充到 `.agents/project-docs-index.md` 文件中
5. 在 `AGENTS.md` 中的"项目文档索引"部分添加引用：
   ```markdown
   ## 项目文档索引
   - 项目文档索引: .agents/project-docs-index.md
   ```

### 步骤 4：处理项目外的本地知识库文档

1. 询问用户："需要索引哪些不在项目内的本地知识库文档？"
2. 用户回答后（例如："不在项目内的知识库文档索引文件路径有：~/docs/index.md, ~/code/index.md, ~/wikis/index.md"）
3. 直接将用户提供的索引文件路径添加到 `AGENTS.md` 中的对应位置：
   ```markdown
   ## 项目外，但在本地的知识库文档索引
   - ~/docs/index.md
   - ~/code/index.md
   - ~/wikis/index.md
   ```

### 步骤 5：创建编码约束文件

创建 `.agents/constraints.md` 文件：

1. 直接创建这个文件
2. 在文件中添加描述："这是项目内的编码约束的文件，包含了有哪些项目内的编码约束"
3. 在 `AGENTS.md` 中的"项目内的编码约束"部分添加引用：
   ```markdown
   ## 项目内的编码约束
   - 项目内的编码约束: .agents/constraints.md
   ```

## 重要提示

- 这个 skill 会创建 `.agents/` 目录，请确保在项目根目录下执行
- 如果 `AGENTS.md` 文件已存在，请先读取其内容，然后增量更新，而不是覆盖整个文件
- 在扫描项目目录结构时，只扫描用户指定的目录，不要扫描整个项目
- 询问用户时，使用简洁明确的问题，并等待用户回答后再继续
